### Задание 1
 Активный master-сервер и пассивный репликационный slave-сервер - основные плюсы этого подхода заключаются в том что на пассивном сервере всегда есть актуальная копия данных,
 с которой можно каким либо образом поработать не затрагивая основной сервер, можно создавать бэкапы, обслуживать базы не нагружая Master. Также при выходе Master из строя можно заменить его Slave, пока выполняются работы по восстановлению основного сервера.
 Репликация Master и несколько Slave - представляет собой выполнение записи на Master сервер, а чтение выполняется с Slave серверов. Таким образом мы снимаем нагрузку с Master сервера и распределяем её по Slave серверам, если у нас есть в этом необходимость(например: операций чтения намного больше чем записи). Также мы можем разнести Slave сервера по разным географическим положениям для более быстрого доступа пользователя к чтению данных. Также при выходе из строя нескольких Slave, система продолжит функционировать распределив нагрузку на оставшиеся сервера.

### Задание 2

<img src = "img/144.png" width = 100%>

Выносим каждую табличку в отдельную базу данных, пользователей переносим на отдельный сервер, для того чтобы при возникновении проблем с сервером или базой пользователей можно было просматривать книг в магазинах в гостевом режиме, без полной остановки работоспособности, а лишь частичной.
На базе Users используем горизонтальное шардирование Master-slave, для того чтобы уменьшить нагрузку на Master, читаем со Slave. В случае выхода из строя Master, перенастраиваем Slave в Master и продолжаем работу, пока не будет восстановленна основная база.
База данных магазинов работает в таком же режиме как и пользователей. Но находится на одном сервере с базой книг и связана с ней по ShopID(id магазина в котором эти книги находятся).В базе данных книг используем горизонтальный шардинг для разбиения таблиц по диапазонам букв. Также используется Master-Slave репликация.
Вертикальное маштабирование применяется при необходимости увеличить мощности на сервере, если это является целесообразным.
