### Задание 1
 Репликация Master-Slave представляет собой выполнение записи на Master сервер, а чтение выполняется с Slave серверов.
 Таким образом мы снимаем нагрузку с Master сервера и распределяем её по Slave серверам, если у нас есть в этом необходимость(например: операций чтения намного больше чем записи).
 Репликация Master- Master представляет собой выполнение и записи, и чтения с обоих серверов. Таким образом мы можем объеденить несколько удалённых офисов
 и гарантировать скорость записи данных на сервер.
 Главные отличия:Master-slave - прост в настройке, отсутствие коллизий(так как данные пишутся на один сервер), простота поддержки, широкие возможности маштабирования.
    Master-Master - сложен в настройке, позволяет читать и писать на всех узлах, сложен в обслуживании, возможны коллизии данных, обеспечивает повышенную доступность к данным.
